real customer data into the repo.

## Payload templates file (local only)

You can store synthetic payloads in a local JSON file and reference it with:

- `FORTE_TEST_PAYLOADS_PATH` (or `_SANDBOX` / `_PRODUCTION`)

Template file:

- `api-demo-php-harness/config/payload-templates.example.json` (tracked)
- `api-demo-php-harness/config/payload-templates.local.json` (local only, gitignored)

Notes:

- Placeholder tokens like `ott_SAMPLE_TOKEN`, `ORDER_NUMBER_SAMPLE`, or `B-AGREEMENT-SAMPLE` must be replaced
  with real values before those tests will run.

## Notes

- Keep production test values in local `.env` only (gitignored).
- Amounts in automated tests should stay at the lowest denominations (0.01 / 0.02).
