James' Importer++

 

This is a REST app for importing 3rd-party data and creating export files with the customer and paymethod tokens. It also creates a csv file of the records that failed, and creates an error log of the reason why they failed.

Excel Template - The import file columns must be in this order.
  1. Massage the raw data and paste it into the Excel template.
  2. Save the data files (without the header row) as "data.CC.csv" and "data.ACH.csv" in the /toolbox/importer folder.
  3. Import the CC and ACH data files into an empty, live mid.
  4. Open "failure.data.csv" and "failure.log.txt" in Notepad++
  5. Fix the failures.
  6. Import the failures.
  7. Export the token files.
  8. Move the data to the target mid using the DE job "Transfer Client Data".
